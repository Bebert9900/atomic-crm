Email sync

Atomic CRM synchronises each commercial’s mailbox over IMAP and attaches the matching messages to the corresponding contacts. The goal is to let every sales rep see the full email history with a prospect directly on the contact page.

Moving parts

Section titled “Moving parts”
  • Tables (migrations 20260416160000_email_sync.sql and 20260416170000_email_sync_vault.sql):
    • email_accounts — IMAP/SMTP credentials per commercial (password encrypted via pgcrypto with a key stored in Supabase Vault under the name email_encryption_key).
    • email_messages — stored emails with contact_id foreign key.
    • email_sync_state — last UID seen per (account, folder).
  • View unread_emails_summary — feeds the dashboard “Emails non lus” widget.
  • Worker scripts/email-sync.mjs — Node script, connects to each active account, pulls INBOX + Sent, matches senders/recipients against contacts.email_jsonb, and upserts the matched messages.
  • UI
    • UnreadEmailsList on the dashboard (under Upcoming Tasks).
    • ContactEmails tab on the contact page (mobile) and below the notes on desktop.
    • EmailAccountsPage settings page under the user menu (admins only).
  • send_email edge function now picks the SMTP config from the caller’s email_accounts row (falls back to the legacy SMTP_* env vars).

Required environment

Section titled “Required environment”

See scripts/email-sync.env.example:

  • SUPABASE_URL — the project URL (https://<ref>.supabase.co).
  • SUPABASE_SERVICE_ROLE_KEY — needed because the script calls decrypt_email_password (service-role only) and writes via PostgREST with RLS bypassed.

Never commit this file and keep it chmod 600.

What the worker does

Section titled “What the worker does”

For each active email_accounts row:

  1. Decrypts the password via decrypt_email_password.
  2. Connects to IMAP (TLS if port 993).
  3. For each folder in INBOX + the server’s Sent folder (multiple aliases tried):
    • Reads messages with UID > last synced UID.
    • Matches any from/to/cc email address against contacts.email_jsonb.
    • Upserts matched messages into email_messages (conflict on (email_account_id, folder, message_id)), linking contact_id.
    • Skips messages that don’t involve any known contact.
  4. Updates email_sync_state.last_uid.

Unmatched emails are intentionally ignored so the table stays scoped to prospect conversations.
